* Build Guide

[[file:./readme.org][Back to Documentation]] | [[file:../zh-CN/构建指南.org][中文版]]

** Recommended Build

zpp is built as an independent ZetaX project. It keeps zpp-specific build logic
in =builder/=, while shared dependency versions and package configs come from
=zeta_forge deps=.

From the zpp checkout:

#+begin_src sh
cd "$ZETAX_ROOT/zeta_forge"
./zbuild.py deps --BUILD_TYPE=Debug --install

cd "$ZETAX_ROOT/zpp"
./zbuild.py --BUILD_TYPE=Debug --rebuild
#+end_src

=ZETAX_ROOT= is the top-level ZetaX workspace root. In the standard layout,
=zpp= and =zeta_forge= are sibling checkouts under it.

zpp does not run Conan. If =spdlog=, =fmt=, =RapidJSON=, or =GTest= package
configs are missing, install the shared environment with
=zeta_forge/zbuild.py deps --install= first.

** Build Modules

By default, zpp builds the core module and examples. Optional modules are enabled
through builder flags:

- =--with-folly=: enable the Folly module and related examples/tests.
- =--with-nng=: enable the NNG module and examples.
- =--with-taskflow=: enable Taskflow examples.
- =--with-hpx-examples=: enable HPX examples.

Example:

#+begin_src sh
./zbuild.py --rebuild --with-folly --with-nng --with-taskflow --with-hpx-examples
#+end_src

** Common Options

- =./zbuild.py --rebuild --no-tests= builds without tests.
- =./zbuild.py --rebuild --no-examples= builds without examples.
- =./zbuild.py --rebuild --install= builds and installs zpp into the configured zeta_forge install prefix.

** Run Entries

zpp can list and execute runnable entries declared with =add_run_target(...)= in
its =CMakeLists.txt= files:

#+begin_src sh
./zbuild.py runs
./zbuild.py run zpp_core
./zbuild.py run --BUILD_TYPE=Debug zpp_core
#+end_src

=runs= prints each run entry and the equivalent CMake command. =run= directly
executes the already built binary with configured arguments for the selected
build type. It does not configure the project or call =cmake --build=.

** Direct CMake Usage

Direct CMake usage is for advanced/manual scenarios only. It must pass explicit
dependency paths because zpp no longer performs fallback auto-discovery for
=ZETA_CMAKE_UTIL_DIR= or =ZETA_DEPS_CMAKE_DIR= inside CMake.

#+begin_src sh
cmake -S . -B build/Debug \
  -DCMAKE_BUILD_TYPE=Debug \
  -DZETA_CMAKE_UTIL_DIR="$ZETAX_ROOT/zeta_forge/cmake_util" \
  -DZETA_DEPS_CMAKE_DIR="$ZETAX_ROOT/zeta_forge/install/lib/cmake/zeta_deps/Debug"

cmake --build build/Debug
#+end_src

The old =cbuild= helper is no longer the recommended entry for zpp because it
may miss the installed zeta_forge dependency package environment.
