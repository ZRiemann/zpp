#+TITLE: zpp Release History

* Release History

** v0.5.1 - 2026-05-13
*** 发布定位摘要
- 修复 zpp 日志短线程号、队列 chunk 生命周期和 taskflow pipeline 运行稳定性。
- 按补丁版本发布，zpp 从 =v0.5.0= 升级到 =v0.5.1=。

*** 变更内容
- =uni_svr= 运行入口改用毫秒级等待并避免输出误导性的错误级别示例日志，执行完成后可自行退出。
- zpp 日志宏输出的 pattern =%t= 复用 =z::tid::id()= 两位短线程号，避免显示过长的系统线程号并保持日志列对齐；直接调用原生 =g_spd_log->log(...)= 仍保留 spdlog 默认线程号语义。
- =spsc/mpmc= 初始化和复用 chunk 时清理 =next= 链指针，避免未扩容队列析构时读取未初始化指针导致堆损坏。
- =pipeline_mpsc= 恢复复用 =z::mpmc=，并在 =wait_done()= 中等待 Taskflow run callback 完全退出，避免局部 pipeline 提前析构。
- zpp CMake project version 与仓库根 =VERSION= 升级到 =v0.5.1=。

** v0.5.0 - 2026-05-13
*** 发布定位摘要
- 新增项目运行入口 CLI，统一列出并执行 =add_run_target= 预置命令。
- 按功能版本发布，zpp 从 =v0.4.1= 升级到 =v0.5.0=。

*** 变更内容
- =zbuild.py= 新增 =runs= / =run <name>= 子命令，复用 =zeta_forge.run_targets= 扫描并执行项目内 =add_run_target(...)= 运行入口。
- =runs= 输出运行入口名称和对应 CMake 命令，保持列表简洁可读。
- =run <name>= 直接执行已有 build tree 中已经编译好的二进制文件及其预置参数，不自动 configure，也不调用 =cmake --build=。
- zpp CMake project version 与仓库根 =VERSION= 升级到 =v0.5.0=。

** v0.4.1 - 2026-05-13
*** 发布定位摘要
- 优化 zpp 日志宏的关闭级别成本，保持 =spd_inf= 等历史入口兼容。
- 按补丁版本发布，zpp 从 =v0.4.0= 升级到 =v0.4.1=。

*** 变更内容
- =spd_inf/spd_err/spd_war/spd_dbg/spd_trc= 遵守 =SPDLOG_ACTIVE_LEVEL=，用户显式关闭的日志级别会在编译期裁剪。
- 日志宏增加运行时短路判断，关闭级别且未启用 backtrace 时不再求值日志参数。
- =spd2*= printf 日志宏复用轻量级运行时短路，关闭级别时避免参数求值和 =vsnprintf= 格式化。
- =z::server= 日志配置读取直接覆盖 =z::log::config= 默认值，减少临时配置变量并保留非法数值保护。
- 保留 =spd_logx= 薄模板封装和 =g_spd_log->log(...)= 原生 spdlog 热路径。

*** 验证
- 新增日志单元测试覆盖运行时短路、backtrace 缓存保留、printf 短路和 =SPDLOG_ACTIVE_LEVEL= 编译期裁剪场景。

** v0.4.0 - 2026-05-13
*** 发布定位摘要
- 重构 zpp 日志初始化封装，保留原生 spdlog 热路径性能，同时简化初始化与生命周期管理。
- 按接口重构发布，zpp 从 =v0.3.1= 升级到 =v0.4.0=。

*** 变更内容
- 重构 zpp 日志初始化封装：保留 =g_spd_log->log(...)= 原生 spdlog 热路径，新增 =z::log::config/init/shutdown= 负责集中初始化与生命周期管理。
- 删除私有 =spdguard= raw pointer 管理，=z::server= 改为直接通过 =z::log::config= 初始化 console/file/async 日志。
- =<zpp/spd_log.h>= 继续作为 CUDA 友好的 printf 薄封装，不包含 spdlog/fmt 重头文件，并修复 =spd2war= 等级映射和长消息截断处理。
- 移除未使用的历史兼容函数 =z::spd_trace=，=spd2*= 宏统一转发到 =z::log::trace_printf=。
- zpp CMake project version 与仓库根 =VERSION= 升级到 =v0.4.0=。

*** 验证
- 新增 =logging_test.cpp= 覆盖 fallback logger、重复 shutdown、printf warn 等级和长消息截断场景。

** v0.3.1 - 2026-05-11
*** 发布定位摘要
- 完成旧 =zpp/json.h= 到成熟 =zpp/json.hpp= API 的迁移，并硬删除旧头文件。
- 按补丁版本发布，zpp 从 =v0.3.0= 升级到 =v0.3.1=；fortune 项目版本保持不变。

*** 变更内容
- zpp core、nng engine、thread_pool 测试服务中的配置读取改用 =json.hpp= 的 =member=、=get=、=size=、=at= 接口。
- fortune quotations 配置读取改用 =json.hpp=，并将消息队列配置入口类型改为 =z::json_doc*=。
- 删除旧 =include/zpp/json.h=，旧的 =tests/uni_svr/test_json.cpp= 和 =test_json()= 入口同步移除。
- zpp package config 在导入 targets 前恢复 =Threads=、=rapidjson::rapidjson=、=spdlog::spdlog= 等 PUBLIC 依赖目标，降低下游显式依赖顺序要求。
- =zbuild.py --with-hpx-examples= 现在会向 CMake 注入 zeta_forge HPX 的 =AsioConfig.cmake= 兼容路径，避免 =find_package(HPX)= 阶段找不到 Asio。
- folly/nng examples 改为链接 =zpp::folly= / =zpp::zpp_nng= 模块目标，由 zpp 模块传递第三方依赖，避免兄弟目录看不到局部 imported target。

*** 兼容性与升级说明
- 旧 =json.h= 不再保留兼容 shim；应用必须包含 =<zpp/json.hpp>= 并使用新接口。
- 构造/插入临时 JSON 子对象时优先使用右值 =set/push=，需要继续复用源对象时才使用拷贝语义。
- =rapidjson= 仍是 =json.hpp= 的底层依赖，不从 zpp core 依赖中移除。

*** 验证
- 静态扫描确认无 zpp/fortune 直接包含 =zpp/json.h= 的生产代码残留。
- =json_hpp_test.cpp= 继续覆盖解析、构建、move 插入、遍历、删除和非 object JSON 文件加载场景。
- =./zbuild.py --BUILD_TYPE Debug --rebuild= 通过。
- =./build/Debug/tests/unit_tests/gtest_tsc= 通过，13 个测试全部通过。
- =./zbuild.py --BUILD_TYPE Debug --with-nng --no-tests --no-examples --rebuild= 通过。
- =./zbuild.py --rebuild --with-hpx-examples --with-folly --with-nng --with-taskflow= 通过。

** v0.3.0 - 2026-05-11
*** 发布定位摘要
- =z::json= 新增删除接口，公开 API 能力从配置读取/构建扩展到 object member 和 array item 删除。
- 按语义化版本约定，本次接口增加从 =v0.2.x= 升级到 =v0.3.0=。
- 修复 zpp Debug 构建中 Conan imported target 被错误映射到 Release 配置的问题。

*** 变更内容
- =z::json::remove(const std::string &key)=：删除 object 中指定 key 的 member。
- =z::json::remove(size_t index)=：删除 array 中指定 index 的 item，并保持后续元素前移。
- 删除接口错误码约定：
  - 当前 json value 不是对应 object/array 时返回 =ERR_NOT_SUPPORT=。
  - object key 不存在时返回 =ERR_NOT_EXIST=。
  - array index 越界时返回 =ERR_OUT_OF_RANGE=。
- =json_hpp_test.cpp= 新增 object member 删除、array item 删除、missing key、越界和类型不支持场景覆盖。
- =builder/zpp.py= 移除错误的 =CMAKE_MAP_IMPORTED_CONFIG_DEBUG=Release= 全局映射，并在 configure 时清理旧 cache 中的 =CMAKE_MAP_IMPORTED_CONFIG_DEBUG=。
- =builder/zpp.py= 将 =zbuild.py= 和 =builder/zpp.py= 纳入 configure dependency，构建脚本变更后会触发重新 configure。

*** 兼容性与升级说明
- 既有 =set=、=get=、=member=、=at=、=push=、迭代接口保持兼容。
- 新增删除接口不会改变缺失值读取行为，删除后继续通过 =get/member/at= 按既有错误码返回。
- Debug 构建现在消费 Debug Conan 依赖配置，不再依赖 Release imported target 映射。

*** 验证
- =./zbuild.py --BUILD_TYPE Debug= 通过。
- =./build/Debug/tests/unit_tests/gtest_tsc= 通过，13 个测试全部通过。

** v0.2.1 - 2026-05-11
*** 发布定位摘要
- 将 zpp 改造为面向开源用户的轻量默认构建：默认只构建 core 核心模块。
- =folly=、=nng=、=taskflow= 等重依赖改为可选模块，默认关闭。
- 依赖查找下沉到真正编译对应模块的 CMake 子目录中，顶层 =CMakeLists.txt= 不再强制查找。

*** 变更内容
- 新增 CMake 选项：=ZPP_BUILD_FOLLY_MODULE=、=ZPP_BUILD_NNG_MODULE=、=ZPP_BUILD_TASKFLOW_MODULE=（默认 OFF）。
- 顶层 =CMakeLists.txt= 只保留 core 必需依赖：=Threads=、=rapidjson=、=spdlog=。
- 移除顶层强制 =find_package(folly)=、=find_package(nng)=、=find_package(taskflow)=。
- 依赖查找下沉：
  - =folly= 依赖查找移到 =src/folly/CMakeLists.txt=，仅在 =ZPP_BUILD_FOLLY_MODULE=ON= 时执行。
  - =nng= 依赖查找移到 =src/nng/CMakeLists.txt=，仅在 =ZPP_BUILD_NNG_MODULE=ON= 时执行。
  - =taskflow= 依赖查找移到 =examples/taskflow/CMakeLists.txt= 和 =examples/hpx/taskflow/CMakeLists.txt=，仅在 =ZPP_BUILD_TASKFLOW_MODULE=ON= 时执行。
- =src/CMakeLists.txt= 改为条件编译：folly、nng 模块仅在对应选项启用时添加。
- =examples/CMakeLists.txt= 改为条件编译：folly、nng、taskflow examples 按对应选项启用。
- =examples/hpx/CMakeLists.txt= 改为条件编译：HPX taskflow examples 需同时满足 =ZPP_BUILD_HPX_EXAMPLES=ON= 和 =ZPP_BUILD_TASKFLOW_MODULE=ON=。
- =tests/CMakeLists.txt= 改为条件编译：coro 测试（依赖 folly）仅在 =ZPP_BUILD_FOLLY_MODULE=ON= 时编译。
- =tests/unit_tests/CMakeLists.txt= 改为条件编译：folly 单元测试仅在 =ZPP_BUILD_FOLLY_MODULE=ON= 时编译。
- =builder/zpp.py= 新增 CLI 选项：=--with-folly=、=--with-nng=、=--with-taskflow=（默认关闭）。
- =builder/zpp.py= 条件化 CMake 变量：=CMAKE_PREFIX_PATH=、=folly_DIR=、=OpenSSL_DIR=、=TASKFLOW_ROOT= 仅在对应模块启用时注入。

*** 兼容性与升级说明
- Core 默认构建路径：=./zbuild.py --rebuild= 只构建 zpp::zpp 核心库，无需额外依赖（folly/nng/taskflow）。
- 启用可选模块：=./zbuild.py --rebuild --with-folly --with-nng --with-taskflow= 构建所有模块。
- =ZPP_BUILD_EXAMPLES=ON= 不自动启用可选模块；未启用模块的示例自动跳过编译。
- 既有 =ZETA_ZPP_SRC_DIR=、=ZETA_TASKFLOW_SRC_DIR= 等变量覆盖逻辑保持兼容。

*** 验证
- Core 默认构建：=./zbuild.py --BUILD_TYPE=Debug --no-tests --no-examples --rebuild= 通过，不执行 =find_package(folly)=、=find_package(nng)=、=find_package(taskflow)=。
- Folly 单模块：=./zbuild.py --BUILD_TYPE=Debug --with-folly --no-tests --no-examples --rebuild= 通过。
- NNG 单模块：=./zbuild.py --BUILD_TYPE=Debug --with-nng --no-tests --no-examples --rebuild= 通过。
- Taskflow 单模块：=./zbuild.py --BUILD_TYPE=Debug --with-taskflow --no-tests --no-examples --rebuild= 通过。
- 组合模块：=./zbuild.py --BUILD_TYPE=Debug --with-folly --with-nng --with-taskflow --no-tests --no-examples --rebuild= 通过。

** v0.2.0 - 2026-05-11
*** 发布定位摘要
- zpp 从遗留自带 CMake helper/submodule 构建模式迁移为独立 ZetaX 项目。
- zpp 自己维护项目级 =builder/zpp.py= 和 =builder/conanfile.py=，公共构建生命周期复用 =zeta_forge=。
- =cmake_util= submodule 停止作为 zpp 依赖，CMake helper 改由 =$ZETAX_ROOT/zeta_forge/cmake_util= 提供。

*** 变更内容
- 新增轻量 =zbuild.py=，通过 =$ZETAX_ROOT/zeta_forge= 或兄弟目录定位 zeta_forge 公共库。
- 新增 =builder/zpp.py=，继承 =zeta_forge.cmake_builder.CMakeProjectBuilder=，注入 zpp 源码目录、Conan 配方、CMake 选项、测试/示例开关。
- 新增 =builder/conanfile.py=，维护 zpp 项目私有 Conan 依赖。
- 移除 =cmake_util= submodule 和旧 =.gitmodules= 绑定。
- =CMakeLists.txt= 改为通过 =ZETA_CMAKE_UTIL_DIR= 引入公共 CMake helper，并优先使用 zeta_forge 注入的依赖环境。
- Debug zpp 构建可消费 forge 已部署的 Release folly 依赖配置，通过 =CMAKE_MAP_IMPORTED_CONFIG_DEBUG=Release= 映射 imported targets。
- zpp CMake project version 与安装包版本统一为 =0.2.0=，仓库根新增 =VERSION=。

*** 兼容性与升级说明
- 推荐命令为 =cd $ZETAX_ROOT/zpp && ./zbuild.py --rebuild=。
- 旧 =cmake_util= submodule 和 =cbuild= 路径不再作为推荐构建方式维护。
- 构建前应确保 =$ZETAX_ROOT/zeta_forge= 存在，并且基础依赖已按 zeta_forge 工具链准备。
- 如需覆盖源码路径，可继续使用 =ZETA_ZPP_SRC_DIR=、=ZETA_TASKFLOW_SRC_DIR=、=ZETA_RAPIDJSON_SRC_DIR= 等变量。

*** 验证
- =python3 -m py_compile zbuild.py builder/zpp.py= 通过。
- =./zbuild.py --help= 通过。
- =./zbuild.py --BUILD_TYPE=Debug --no-tests --no-examples --rebuild= 通过，完成 Conan install、CMake configure 和静态库构建。
