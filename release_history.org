* Release History

** v0.11.0 - Unreleased
*** Changes
- Extend =z::log::config= with optional structured important file logging while preserving the legacy rotating runtime file fields.
- Add =spd_imp(...)= as an independent important-channel entry point that writes normal =info= records to the runtime log and the important JSONL log without adding an extra =important= JSON field.
- Add JSONL formatting for important logs with stable service, node, run, process/thread, source-location, component, action, and message fields.
- Parse common message prefixes such as =[ctp-md:login-rsp]= and =[nng:socket:create]= into structured =component= and =action= fields.
- Extend =z::server= =spd= JSON parsing with =service=, =node=, =run_id=, and =important= sink settings.
- Rate-limit repeated NNG internal logger bridge records by level, facility, message id, and message so important logs keep the first occurrence and periodic summaries instead of every retry.

*** Verification
- Add logging tests for warn+ JSONL important sinks, direct =spd_imp(...)= info records, legacy runtime file compatibility, and async preservation of the zpp short thread id.
- Add NNG runtime logging coverage that repeated internal records are suppressed while different levels or messages still pass through.

** v0.10.0 - 2026-07-12
*** Changes
- Rename the HPX wrapper namespace from =z::zhpx= to =z::hpx= and keep the public wrapper surface aligned under =zpp/hpx/= and =z::hpx::exec=.
- Add =z::hpx::exec= wrapper exports for sender/receiver concepts, factories, schedulers, algorithms, adapters, and lifecycle helpers as thin forwards over HPX/stdexec compatibility bindings.
- Add explicit performance warnings in HPX exec wrapper docs for frequent future/sender boundary conversions (for example =from_future()= and =into_future()=) to discourage per-message or per-small-task conversion on hot paths.
- Add explicit performance warnings for unnecessary scheduler switches (=starts_on=, =continues_on=, =on=, =transfer=) and overly fine-grained task decomposition with =schedule()= and tiny =then()= stages.

*** Verification
- Keep wrapper APIs intentionally thin and behaviorally equivalent to the underlying HPX/stdexec semantics while improving call-site guidance for performance-sensitive pipelines.

** v0.9.1 - 2026-07-10

- Ignore expected =NNG_ETIMEDOUT= receive results in =replier::on_receive()= to avoid emitting error logs for timeout-driven polling loops.

*** Verification
- Keep NNG socket and request/reply behavior unchanged while improving runtime observability and reducing timeout log noise.

** v0.9.0 - 2026-07-10

- Refactor =pub_sub_device= to a two-phase lifecycle of =configure() -> start() -> stop()= and allow both ingress and egress endpoint sets to mix listen and dial roles.
- Update the pub/sub device example and NNG example documentation to match the new endpoint and lifecycle model.

*** Verification

** v0.8.0 - 2026-07-09
*** Changes
- Add the transport-independent =z::wire::writer=, =reader=, and =size_counter= APIs under =zpp/wire/= for bounds-checked fixed-buffer encoding, symmetric decoding, and encoded-size calculation through matching operation sequences.
- Optimize little-endian scalar reads and writes with fixed-size =memcpy= on native little-endian systems and a shared byte-swap conversion on big-endian systems.

*** Verification
- Add focused unit coverage for byte order, writer/reader completion, writer/size-counter parity, accounting, truncated input, and overflow without partial operations.

** v0.7.0 - 2026-07-05
*** Release Positioning Summary
- Feature release from =v0.6.0= to =v0.7.0=, centered on reusable NNG protocol wrappers and repository-wide C++ formatting.
- Breaking API cleanup in =zpp::nng= removes obsolete compatibility surfaces and standardizes protocol lifecycle and ownership semantics.

*** Changes
- Add a repository =.clang-format= configuration and VS Code formatter settings, then normalize zpp C++ sources, headers, examples, and tests to the shared format.
- Unify public protocol wrappers under =zpp/nng/protocols/= and add the =protocols.h= aggregate header; the former =zpp/nng/publisher.h= and =zpp/nng/subscriber.h= include paths are removed without forwarding headers.
- Add reusable =endpoint_protocol= lifecycle support and complete PUSH/PULL, REQ/REP, SURVEYOR/RESPONDENT, BUS, and single-endpoint PAIR0/PAIR1 wrappers; REQ/REP and SUB use managed asynchronous concurrency while the remaining protocols expose synchronous and AIO operations in each legal direction.
- Upgrade =aio_ctx= to movable transactional RAII with only asynchronous =send(message&)= and =recv()= operations; REQ/REP manage their contexts internally, while SURVEYOR and RESPONDENT retain application-managed contexts with protocol-specific survey options.
- Add the thin =z::nng::context= wrapper for =nng_ctx= lifecycle, synchronous/AIO transfers, current NNG 2.0 typed context options, and batch configuration of common send/receive timeouts, and compose it into =aio_ctx=.
- Add reusable =publisher= and =subscriber= wrappers with transactional multi-endpoint configuration, optional socket/transport/listener/dialer option groups, mixed listen/dial roles, and caller-managed lifecycle sequencing; subscriber manages a fixed pool of receive AIOs on the default context so each matching message is delivered once.
- Replace the broad =event_forwarder= API with =pub_sub_device= under =zpp/nng/protocols/=, use the RAII =aio= wrapper for device completion, relinquish sockets after NNG takes ownership, and remove the redundant =running()= state query.
- Keep listener/dialer endpoint creation and startup as private =endpoint_protocol= implementation details shared by all protocol wrappers.
- Move endpoint metadata validation into =zpp/nng/endpoint.h= and complete batch =set_options/get_options= APIs for sockets, listeners, and dialers, including independently optional endpoint and TCP transport groups.
- Inline synchronous PUB send forwarding for the hot path while preserving NNG result codes and explicit message ownership semantics.
- Rename the PUB/SUB programs to =publisher_example= and =subscriber_example=, add =pub_sub_device_example=, and demonstrate a three-process publisher-to-device-to-subscriber forwarding topology.
- Add runnable PUSH/PULL, REQ/REP, and PAIR1 listen/dial examples with shared JSON option parsing, reusable AIO state machines, and matching =doc/server.json= sections.
- Unify REQ/REP lifecycle under =start(context_count)= and =stop()=: startup creates managed concurrent contexts and shutdown closes AIO/context resources before endpoints. Remove public =open_context()/send()/recv()= and =start_mt()/stop_mt()=; requester correlates out-of-order replies through =request()/on_reply()= hints.
- Unify SUB lifecycle under =start(aio_count)= and =stop()=: startup posts multiple receives on the socket's single default context, callback completion automatically resubmits its AIO, and shutdown stops all AIOs before endpoints. Remove public =subscribe()/unsubscribe()/open_context()/recv()= and deliver messages through the concurrent =on_receive(message&, result)= extension point.
- In =zpp/nng/pipe.h=, remove pointer overloads for =get_bool/get_int/get_ms/get_size/get_string/get_strdup/get_strlen= inside the Pipe Options region.
- Keep and normalize reference overloads so each method directly calls the underlying =nng_pipe_get_*= API using the referenced output variable.
- Keep =get_strcpy(const char*, char*, std::size_t)= unchanged because it is a buffer-oriented API that is naturally pointer-based.
- Follow NNG 2.0 by removing =dialer/listener::set_uint64/get_uint64= and =dialer::get_addr=, whose upstream implementations and exported symbols no longer exist; =dialer::set_addr= remains available.
- Simplify =nng::message= ownership so a non-null =msg_= is always owned and successful sends or =release()= leave the wrapper empty, removing the separate =owned_= state.
- Add stage-specific protocol framework error logs for configuration, endpoint startup, managed AIO/context creation, callback-state violations, and requester context-pool initialization or recycle failures; expected backpressure and orderly shutdown remain non-error paths.

*** Compatibility and Upgrade Notes
- Update PUB/SUB includes to =zpp/nng/protocols/publisher.h= and =zpp/nng/protocols/subscriber.h=, or include =zpp/nng/protocols/protocols.h= for the complete protocol surface.
- Manage REQ/REP through =configure() -> start(context_count) -> request/serve -> stop()= instead of separate =start_mt()/stop_mt()= calls. SURVEYOR and RESPONDENT application-managed contexts must be closed before =stop()=.
- Manage SUB through =configure() -> start(aio_count) -> receive callbacks -> stop()=. Migrate direct =subscribe()/unsubscribe()/open_context()/recv()= use to configured topics and =on_receive()=, or use =z::nng::socket=, =context=, and =aio_ctx= directly when low-level operations are required.
- Migrate former REQ/REP =open_context()/send()/recv()= calls to requester =request()/on_reply()= or the replier =on_receive()/on_send()= extension points; use =z::nng::context= and =aio_ctx= directly for lower-level context operations.
- Replace removed synchronous or raw-message =aio_ctx= send/receive calls with =context= for synchronous I/O or =aio_ctx::send(message&)= and =recv()= for asynchronous I/O. Asynchronous send transfers ownership when submitted; recover a failed send with =io().release_msg()=. Access the context handle and id through =ctx().get()= and =ctx().id()= because the duplicate =aio_ctx::get()/id()= forwarding helpers were removed.
- Replace =event_forwarder= and its header with =pub_sub_device= from =zpp/nng/protocols/pub_sub_device.h=; the old type, include path, and =running()= method have no compatibility aliases.
- PUB/SUB example configurations must provide one =ingress_endpoint=, a non-empty =egress_endpoints= array, and a non-empty =topics= array; an empty topic string subscribes to all messages.
- Replace calls to the removed =socket::apply_options()= helper with =socket::set_options()= and handle its returned NNG error code.
- This is a source-level breaking change for callers that passed explicit output pointers to the removed =pipe::get_*= overloads.
- Migration pattern: replace pointer calls with reference calls, for example =pipe.get_int(name, value)= instead of =pipe.get_int(name, &value)=.
- Remove calls to the deleted dialer/listener option wrappers; rebuilding NNG cannot restore them because current NNG only retains stale declarations for these removed APIs.

** v0.2.1 - 2026-05-11
*** Release Positioning Summary
- Adapt zpp into a lightweight default build for open-source users: only core builds by default.
- Heavy dependencies such as =folly=, =nng=, and =taskflow= become optional modules and are disabled by default.
- Dependency lookup moves down into the CMake subdirectories that actually compile each module; the top-level =CMakeLists.txt= no longer forces those packages to be found.

*** Changes
- Add CMake options =ZPP_BUILD_FOLLY_MODULE=, =ZPP_BUILD_NNG_MODULE=, and =ZPP_BUILD_TASKFLOW_MODULE=, all defaulting to OFF.
- Top-level =CMakeLists.txt= keeps only core required dependencies: =Threads=, =rapidjson=, and =spdlog=.
- Remove top-level forced =find_package(folly)=, =find_package(nng)=, and =find_package(taskflow)= calls.
- Move dependency lookup:
  - =folly= lookup moves to =src/folly/CMakeLists.txt= and runs only when =ZPP_BUILD_FOLLY_MODULE=ON=.
  - =nng= lookup moves to =src/nng/CMakeLists.txt= and runs only when =ZPP_BUILD_NNG_MODULE=ON=.
  - =taskflow= lookup moves to =examples/taskflow/CMakeLists.txt= and =examples/hpx/taskflow/CMakeLists.txt=, and runs only when =ZPP_BUILD_TASKFLOW_MODULE=ON=.
- =src/CMakeLists.txt= now adds Folly and NNG modules conditionally.
- =examples/CMakeLists.txt= now adds Folly, NNG, and Taskflow examples conditionally.
- =examples/hpx/CMakeLists.txt= now builds HPX Taskflow examples only when both =ZPP_BUILD_HPX_EXAMPLES=ON= and =ZPP_BUILD_TASKFLOW_MODULE=ON=.
- =tests/CMakeLists.txt= now builds coroutine tests, which depend on Folly, only when =ZPP_BUILD_FOLLY_MODULE=ON=.
- =tests/unit_tests/CMakeLists.txt= now builds Folly unit tests only when =ZPP_BUILD_FOLLY_MODULE=ON=.
- Add =--with-folly=, =--with-nng=, and =--with-taskflow= CLI options to =builder/zpp.py=, all disabled by default.
- =builder/zpp.py= now conditionally injects =CMAKE_PREFIX_PATH=, =folly_DIR=, =OpenSSL_DIR=, and =TASKFLOW_ROOT= only when the corresponding module is enabled.

- Core default build path: =./zbuild.py --rebuild= builds only the =zpp::zpp= core library without extra dependencies such as Folly, NNG, or Taskflow.

- =ZPP_BUILD_EXAMPLES=ON= does not automatically enable optional modules; examples for disabled modules are skipped automatically.
